Using Dgraph to store DCAT datasets

DCAT

The Data Catalog Vocabulary (DCAT) is an RDF vocabulary to describe datasets, data services and available resources in catalogs and portals. DCAT is a W3C Recommendation and an updated version (2) has been published in February 2020.

DCAT defines a set of classes and properties, and incorporates properties from existing vocabularies such as the Dublin Core vocabulary and FOAF. The figure below (taken from the W3C Recommendation) gives an overview of all the classes, properties and relations of the DCAT model:

Dgraph Types

Dgraph supports a type system that I want to use to encode the different DCAT classes and properties. To do so have to look into Dgraph’s different schema types:

  • Dgraph supports a set of scalar types, e.g. string, int, float, bool, dateTime
  • To further categorize nodes we can define own types; we will use this to define the types Dataset and Distribution.
  • There is no inheritance, no support of sub-properties and sub-types. This means we cannot define an abstract “dcat:Resouce” type as it is present in the DCAT model.
  • We cannot set the range of a property: In Dgraph we can use the uid type to denote a node-node edge, however, we cannot set the type of the target node.

Dgraph does not support namespaces. We encode this information by using the namespace as a prefix (separated by a period). The types in Dgraph are defined using a GraphQL-like syntax. Below you can see the type definitions for Dataset and Distribution; the full model can be found here.

Insert a DCAT Dataset

Dgraph does not directly support to load any RDF into the database. It creates a UID for each blank node used in the input. Also, it does not allow external IDs as node identifiers. The Dgraph documentation recommends to add an additional xid edge and store the external IDs and types as strings:

<0x321> <xid> "https://europeandataportal.eu/set/data/13347329-bundesamt-fur-statistik-bfs" .
<0x321> <http://www.w3.org/1999/02/22-rdf-syntax-ns#type> <0x123> .
<0x321> <dgraph.type> "Dataset" .

I also encoded the external IDs in the prefixes of the types and the properties, as you can see in the above schema. To insert the above RDF example, I have to simplify the input a bit:

  • Replacing all (external) subjects with newly introduced blank nodes
  • Adding a source edge to store these external links
  • Converting all remaining RDF object resources (<http://..>) to strings (“http://..”)

To insert triples in Dgraph we have to use the set keyword:

curl -H "Content-Type: application/rdf" -X POST localhost:8080/mutate?commitNow=true -d $'
{
  set {
_:dataset <source> "https://europeandataportal.eu/set/data/13347329-bundesamt-fur-statistik-bfs" .
_:dataset <dgraph.type> "dcat.Dataset" .
_:dataset <dct.spatial> "Kantone" .
_:dataset <dct.temporal> _:B89a1f708d00f5dad999618a00f47c9c3 .
_:dataset <dcat.keyword> "bevolkerung"@de .
_:dataset <dcat.keyword> "tabella"@it .
_:dataset <dcat.distribution> _:8273cee06700443ab201bcb537ce70b1 .
_:8273cee06700443ab201bcb537ce70b1 <dgraph.type> "dcat.Distribution" .
_:8273cee06700443ab201bcb537ce70b1 <dcat.accessURL> "https://www.bfs.admin.ch/asset/fr/px-x-0102020203_103" .
_:8273cee06700443ab201bcb537ce70b1 <dct.rights> "NonCommercialAllowed-CommercialWithPermission-ReferenceRequired" .
_:8273cee06700443ab201bcb537ce70b1 <dct.issued> "2020-06-25T02:00:00"^^<xs:dateTime> .
_:8273cee06700443ab201bcb537ce70b1 <dcat.mediaType> "text/html" .
 }
}
