Introduction

  • ``Knowledge graph'' in the literature since (at least) 1972
  • Modern incarnation stems from Google Knowledge Graph (2012)
  • Use graphs to represent data, enhanced with some way to explicitly represent knowledge

Knowledge graph

Knowledge graph
accumulates and conveys knowledge of the real world
nodes represent entities, edges represent relations between these entities
conforms to a graph-based data model, e.g. directed edge-labelled graph, property graph, etc.

In practice

  • Open knowledge graphs
  • Enterprise knowledge graphs

Open knowledge graphs

  • Content accessible for the public good
  • Either extracted from Wikipedia, or built by communities of volunteers
  • Cover many domains
  • Most prominent: DBpedia, Wikidata, YAGO

Enterprise knowledge graphs

  • Typically internal to a company
  • Applied for commercial use-cases
  • Web search (e.g., Bing, Google), commerce (e.g., Airbnb, Amazon), social networks (e.g., Facebook, LinkedIn), etc.

Data Graphs

  • Graph-structured data models
  • Graph query languages

Models

  • Consider example data about attractions, events, services, etc.
  • Initial tabular / relational schema with five columns:

Event(name, venue, type, start, end)

  • Generate identifiers for events and adapt relational schema:

EventName(id,name), EventStart(id,start), EventEnd(id,end), EventVenue(id,venue), EventType(id,type)

Directed edge-labelled graphs

  • Multi-relational graph
  • Set of nodes, e.g.: Santiago, Arica, EID16, 2018-03-22 12:00
  • Set of directed labelled edges, e.g.: Santa LucíacitySantiago

Example

Resource Description Framework (RDF)

  • W3C recommendation
  • Different types of nodes: (cf. Identity)
    • Internationalized Resource Identifiers (IRIs)
    • Literals and datatype values
    • Blank nodes

Heterogeneous graphs

Property graphs

  • property–value pairs associated with both nodes and edges
  • used in popular graph databases (e.g. Neo4j)
  • can be translated to/from directed edge-labelled graphs
